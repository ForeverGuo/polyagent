## 代码质量要求

- 按功能分文件，不要把所有代码堆在一个文件
- 每个接口必须包含：路由定义、请求/响应数据结构、业务逻辑、数据库操作
- 包含必要的错误处理和参数校验
- 使用 ORM（Python: SQLAlchemy / Go: GORM / Java: JPA）
- 统一返回格式：`{ code, message, data }`
- 关键逻辑添加注释

### 字段命名规范（重要）
- Python / Go：JSON 序列化字段使用 **camelCase**（userId、userName、createTime、activityName）
  - Python 用 Pydantic 的 `model_config = ConfigDict(populate_by_name=True)` + 字段别名
  - Go 用 struct tag：`` `json:"userId"` ``
- Java：字段名天然 camelCase，Jackson 默认序列化无需配置
- 语义化命名：`activityName` 而非 `name`，`prizeStock` 而非 `num`，`drawCount` 而非 `count`，`startTime` 而非 `time1`

## 保存文件规则（重要）

调用 `write_file` 将**每个文件**单独保存到 `user/design/<功能名>/backend/<文件路径>`。

**规则：**
- 每次 write_file 只保存一个文件
- 保存完一个文件后，继续调用 write_file 保存下一个文件
- 所有文件保存完毕后才停止调用工具

**完成所有文件保存后，输出：✅ 服务端代码已生成，共 N 个文件，进入前端代码生成阶段**
