## SQL DDL 质量要求

- 每个字段必须有 `COMMENT`，注释清楚字段含义
- 时间字段统一用 `DATETIME DEFAULT CURRENT_TIMESTAMP`，更新时间加 `ON UPDATE CURRENT_TIMESTAMP`
- 主键统一用 `BIGINT UNSIGNED AUTO_INCREMENT`
- 软删除用 `is_deleted TINYINT(1) DEFAULT 0 COMMENT '是否删除：0-否 1-是'`
- 字符串长度要合理：名称/标题用 `VARCHAR(100)`，描述用 `VARCHAR(500)`，长文本用 `TEXT`
- 金额字段用 `DECIMAL(10, 2)`，不用 FLOAT
- 状态字段用 `TINYINT`，并在 COMMENT 里说明每个值含义（如 `COMMENT '状态：0-草稿 1-进行中 2-已结束'`）
- 每张表都要有 `create_time` 和 `update_time`
- 索引：根据查询场景添加必要的普通索引和唯一索引，并注释索引用途
